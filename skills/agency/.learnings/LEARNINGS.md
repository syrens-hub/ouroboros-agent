## 经验记录

### [LRN-20260406-001]
- **场景**: Agent 任务交付质量参差不齐
- **问题**: 任务描述模糊导致 Agent 理解偏差
- **做法**: 制定标准化任务模板，明确验收标准
- **效果**: 交付质量显著提升
- **教训**: 任务定义标准化是质量基础

### [LRN-20260406-002]
- **场景**: 多 Agent 协作时接口定义不清晰
- **问题**: Agent 之间输出格式不统一，汇总困难
- **做法**: 建立结构化输出规范，统一接口格式
- **效果**: 协作效率提升
- **教训**: 协作接口标准化先于执行

### [LRN-20260406-003]
- **场景**: Agent 选型与任务不匹配
- **问题**: 选了不合适的 Agent 导致返工
- **做法**: 建立任务类型→Agent映射表，用 task-parser.sh 辅助
- **效果**: 一次选对率提升
- **教训**: 选型决策需要工具辅助

### [LRN-20260406-004]
- **场景**: Agent 执行过程缺乏监督
- **问题**: 任务放出去不管，交付失控
- **做法**: 分段交付 + 中途检查点，用 monitor.sh 监控
- **效果**: 问题早发现早干预
- **教训**: 放权不等于放手

### [LRN-20260406-005]
- **场景**: 交付物验收流于形式
- **问题**: 没有明确标准，好坏凭感觉
- **做法**: 制定验收清单，不合格坚决退回
- **效果**: 交付质量提升
- **教训**: 无标准的验收等于没验收
