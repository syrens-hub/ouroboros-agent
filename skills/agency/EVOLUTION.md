# Agency Skill 自我进化规则

> 让 Agency Agent 团队编排能力持续进化

## 快速触发

| 触发条件 | 动作 |
|---------|------|
| 任务交付质量差 | 记录到 ERRORS.md，分析根因 |
| Agent 选型错误 | 记录到 LEARNINGS.md，更新选型规则 |
| 用户修正了 Agent 行为 | 记录到 LEARNINGS.md，提炼规则 |
| 发现新协作模式 | 记录到 FEATURE_REQUESTS.md |
| Quiz 发现薄弱项 | 更新薄弱项追踪，加强相关规则 |

---

## Review → Apply → Report 循环

### 任务执行前
1. 读取 `.learnings/LEARNINGS.md` 中相关的经验
2. 读取 `.learnings/ERRORS.md` 中相关的错误教训
3. 对照当前任务类型，筛选适用的规则
4. 在任务指令中备注：本次参考了 [LRN-xxx]

### 任务执行中
- 应用已学习的规则
- 如发现规则不适用，记录为"规则例外"
- 使用 monitor.sh 记录检查点

### 任务执行后
1. 总结本次使用了哪些学习
2. 评估效果：规则是否有效
3. 如有新经验，写入 LEARNINGS.md
4. 如有新错误，写入 ERRORS.md

### 用户反馈处理
- 用户修正了 Agent 行为 → 立即记录到 LEARNINGS.md
- 用户抱怨质量 → 对照验收清单，分析哪个环节出了问题
- 同一类问题出现 2 次 → 触发规则更新

## 检测触发条件

### 任务质量差
- **检测**: 交付物对照验收标准有 2 项以上不通过
- **动作**: 记录到 ERRORS.md，触发任务模板检查

### Agent 选型不匹配
- **检测**: 任务中途更换 Agent，或交付物明显不是该 Agent 专长
- **动作**: 记录到 ERRORS.md，更新 task-parser.sh 规则

### 多 Agent 协作失败
- **检测**: 接口不匹配、输出格式不统一、汇总困难
- **动作**: 记录到 ERRORS.md，更新协作接口规范

### 任务分解不足
- **检测**: L3 任务没拆解直接下发，导致 Agent 超载
- **动作**: 记录到 ERRORS.md，强制 L3 任务走 Research 流程

### 用户修正模式
- **检测**: 用户说了"不对"、"不是这样"、"改一下"
- **动作**: 记录修正内容到 LEARNINGS.md，提炼规则

---

## 规则晋升

当同一规则出现 3 次以上，且每次都有效时：

1. 从 LEARNINGS.md 晋升到 `agency-best-practices.md`
2. 从 ERRORS.md 晋升到预防规则
3. 更新 SKILL.md 中的协作原则
4. 更新 task-parser.sh 中的匹配规则

---

## 薄弱项追踪

从 Quiz 或用户反馈中发现薄弱项：

1. 记录到 `learnings/weak-points.md`
2. 制定针对性强化计划
3. 下次相关任务中重点验证
4. 2 周后再次 Quiz，确认改进
